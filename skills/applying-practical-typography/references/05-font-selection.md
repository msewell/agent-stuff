## Font Basics

For body text, acquire these core styles: roman, italic, bold, bold italic,
and roman small caps. Microsoft Office on Windows works better with TrueType
(.ttf) fonts than OpenType (.otf).

Use professional fonts over system fonts whenever possible—they are the
fastest, most visible typographic upgrade.

## System Fonts

System fonts are pre-installed on your computer. Three problems in print:
many are poorly designed, many are optimized for screen (not print), and all
are overexposed.

**Tolerable for print (A list):** Athelas, Bell MT, Californian FB, Calisto
MT, Century Schoolbook, Charter, Garamond, Gill Sans, Goudy Old Style,
Helvetica, Helvetica Neue, Hoefler Text, Iowan Old Style, Optima, Palatino,
Seravek, Sitka.

**OK in limited doses (B list):** Calibri, Cambria, Candara, Constantia,
Corbel, Futura, High Tower Text, Perpetua, Segoe UI.

**Questionable (C list):** Baskerville, Georgia, Times New Roman, Courier,
Courier New, Consolas, Impact, Lucida variants.

**Fatal to credibility (F list):** Arial, Comic Sans, Papyrus, Copperplate,
Bookman Old Style, American Typewriter, Brush Script, Zapfino, Verdana,
Trebuchet MS, Tahoma, and most script/novelty fonts.

Use A-list for print. A and B lists are fine for screen. Avoid C list if
possible. Never use F list in professional writing.

For screen-only work: the legibility advantage of screen-optimized fonts
(Georgia, Calibri) has largely evaporated with modern high-resolution
displays. Use whatever font you'd prefer on the printed page.

## Bad Fonts

Avoid: Papyrus (fake-historic), Bookman (outdated), Bodoni (flashy but
fatiguing for body text), Copperplate (overused and dated), and early digital
script fonts (clunky compared to modern OpenType scripts).

## Goofy Fonts

Novelty, script, handwriting, and circus fonts have no place in professional
documents. Save them for signage or breakfast-cereal boxes.

## Monospaced Fonts

Do not use monospaced fonts for body text. They are harder to read and waste
horizontal space. They exist for typewriter compatibility and code display.

Use monospaced fonts only for: software code, terminal output. If you need
one, avoid Courier—use Consolas or a better alternative.

For aligned numbers in proportional text, use tabular figures (most
proportional fonts include them) rather than switching to a monospaced font.

## Mixing Fonts

1. Mixing is optional, not required. One font with variations (size, bold,
   italic, small caps) provides plenty of range.
2. Rule of diminishing returns: two fonts is fine, three is risky, four or
   more is almost always too many.
3. You can mix any two identifiably different fonts—the "serif + sans serif
   only" rule is a myth. Lower contrast between fonts can be more effective.
4. Give each font a consistent role (one for body, one for headings; or one
   for center content, one for margins).
5. Avoid multiple fonts within a single paragraph.
6. Reliable method: combine fonts by the same type designer.

## Alternate Figures

Fonts may include multiple figure (numeral) styles:

- **Lining figures:** Uniform height, aligned at top and bottom. Best for all
  caps. Most common default.
- **Oldstyle figures:** Variable height, like lowercase letters. Best in
  lowercase body text. Not with caps.
- **Tabular figures:** Fixed width—digits align in vertical columns. Essential
  for grids of numbers and spreadsheets.
- **Proportional figures:** Variable width—better for body text.

Default figures are always acceptable. Alternates are accessed via OpenType
features. To test for tabular figures: type a row of zeros above a row of
ones—if they're the same length, the font has tabular figures.
